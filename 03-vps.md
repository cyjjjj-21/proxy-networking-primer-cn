# VPS：云服务器、合规和被墙

## VPS 是什么

VPS 是租来的远程 Linux 机器。在这个示例架构里，入口 VPS 负责：

```text
在公网 443 端口接收代理连接
运行 Xray
按规则决定流量往哪里出去
```

相关：[[05-xray-routing|Xray：入站、出站、路由和链式代理]]

## VPS 厂商为什么能合法存在

VPS 是中性基础设施，类似机房、宽带、域名、CDN、对象存储。

正常用途：

```text
网站托管
API 服务
开发测试
企业 VPN
游戏服务器
监控任务
备份
自动化脚本
```

可能被滥用：

```text
垃圾邮件
DDoS
钓鱼网站
恶意扫描
欺诈流量
违规代理
```

靠谱 VPS 厂商靠合规和滥用处理机制存活：

```text
服务条款 / AUP
实名、支付、账号记录
Abuse Desk
自动风控
下架和封禁
配合合法请求
制裁和出口管制
```

它们卖的是计算资源，不是内容分发平台。收到有效投诉或法律要求后必须处理。

## Bulletproof Hosting

如果服务商宣传无视投诉、不处理 Abuse、不配合执法、专门接灰黑产客户，这类更接近 Bulletproof Hosting。

风险：

```text
被上游机房清退
被运营商断网
IP 段进黑名单
支付通道被封
被执法打击
```

靠谱 VPS 厂商会努力远离这种标签，因为 ASN/IP 段信誉会影响正常客户。

## VPS IP 被墙是什么意思

VPS IP 被墙，可能是网络环境、平台或监管系统对 IP、端口、协议特征、域名、SNI 或流量模式做了阻断、限速、干扰或风控。

不一定代表用户自己做了坏事。

常见原因：

```text
IP 段历史差
前任用户作恶
整个机房/ASN 被重点关注
协议特征暴露
流量行为异常
目标平台风控
地区网络策略
```

换 IP 是否有用：

```text
单个 IP 坏了 -> 可能有用
整个 IP 段 / ASN 坏了 -> 同厂商换 IP 可能没用
协议特征暴露 -> 换 IP 后还可能再出问题
目标网站拒绝数据中心 IP -> 普通 VPS IP 仍然不好
```

Reality 降低入口代理特征暴露；静态 ISP 代理让目标网站看到更干净的 ISP 型出口。

## VPS 不是匿名黑洞

VPS 厂商通常能持有：

```text
账号信息
付款记录
登录 IP
实例 IP
工单记录
流量统计
Abuse 投诉记录
```

VPS 是信任点，不是绝对隐私屏障。它能帮用户对目标网站隐藏本地真实 IP，但不能保证 VPS 厂商完全不知道用户是谁。
