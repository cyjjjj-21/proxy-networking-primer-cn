# 私域网络：用 VPS 连接自己的设备

代理上网和私域网络是两件事。

代理上网解决的是：

```text
设备访问外网时，目标网站看到哪个出口 IP？
DNS 从哪里解析？
哪些流量走代理，哪些流量直连？
```

私域网络解决的是：

```text
自己的设备如何安全互访？
人在外面时，如何访问家里的电脑、小主机或内网服务？
如何不把家庭路由器端口直接暴露到公网？
```

一句话：

```text
代理上网 = 设备 -> VPS/代理链 -> Internet
私域互联 = 设备 -> VPS/组网中枢 -> 自己的其他设备
```

两者可以共用同一台 VPS，但应该使用不同端口、不同网段、不同路由规则来隔离。

## 为什么 VPS 适合做公网锚点

家庭或办公室里的设备通常在 NAT 后面：

```text
外部网络 -> 家庭路由器 -> 内网设备
```

这条方向很难直接打进来，因为中间可能有：

```text
家庭路由器 NAT
系统防火墙
动态公网 IP
运营商 CGNAT
公共 Wi-Fi 客户端隔离
```

但反过来很容易：

```text
内网设备 -> 路由器 -> Internet -> VPS
```

普通上网就是这个方向。私域网络利用的就是这个事实：让每台设备都主动连到 VPS，再由 VPS 把这些设备放进同一个虚拟内网。

## 示例架构

```text
                 入口 VPS / 组网中枢
                    10.8.0.1
               /        |         \
              /         |          \
家庭小主机          笔记本电脑        手机
10.8.0.2        10.8.0.3       10.8.0.4
```

每台设备都和 VPS 维持一条在线边：

```text
家庭小主机 <-> VPS
笔记本电脑 <-> VPS
手机       <-> VPS
```

谁在线，谁就在这张私域网里；谁断线，谁就从图里消失。

长期稳定的骨架通常是：

```text
VPS <-> 家庭小主机
```

家庭小主机固定插电、固定联网，适合做 24/7 常驻节点。笔记本电脑更像临时加入的工作节点；睡眠、合盖或断网后，其他设备就不能通过私域网访问它。

## WireGuard 和 Tailscale

常见做法有两类：

```text
WireGuard:
自己在 VPS 上搭组网服务。控制权更清楚，配置更手动。

Tailscale:
基于 WireGuard 的组网产品。账号、设备发现和打洞更省心，但依赖第三方控制面。
```

WireGuard 的基本安全模型是：

```text
每台设备都有一对公私钥
VPS 只接受白名单里的 public key
流量在隧道里加密
丢设备后删除对应 peer
```

VPS 可以看见哪些 peer 连上来、连接时间、流量大小，但不能直接读取 WireGuard 隧道里的应用层内容。

## 和代理链路怎么隔离

同一台 VPS 可以同时跑代理入口和私域网络：

```text
Xray Reality:
443/tcp
用途：代理上网入口

WireGuard:
51820/udp
用途：私域设备互访
```

它们进入不同端口、不同进程、不同路由逻辑。

推荐边界：

```text
Xray / Clash / Shadowrocket:
负责公网代理、DNS、防泄漏、出口选择。

WireGuard:
只负责 10.8.0.0/24 私域互访。
不接管默认路由。
不接管 DNS。
不做 Internet 出口。
```

WireGuard 客户端配置里，私域互访场景通常只需要：

```ini
AllowedIPs = 10.8.0.0/24
```

这表示：只有访问 `10.8.0.x` 这些私域地址时才走 WireGuard。

不要在不理解后果时写：

```ini
AllowedIPs = 0.0.0.0/0
DNS = 1.1.1.1
```

`0.0.0.0/0` 表示全部 IPv4 流量都走 WireGuard，会把它变成全局 VPN。这可能干扰原有代理出口、DNS 防泄漏和分流规则。

初期更稳的策略是：

```text
代理上网交给代理系统
私域互联交给 WireGuard
两者互不抢默认路由
```

## 可以访问哪些东西

私域网建立后，移动设备可以通过虚拟内网 IP 访问家庭小主机或笔记本电脑：

```text
ssh user@10.8.0.2
vnc://10.8.0.2
http://10.8.0.2:3000
smb://10.8.0.2
```

常见用途：

```text
远程 SSH
远程桌面
访问本地开发服务器
访问文件共享
查看日志和任务状态
重启服务
远程处理图形界面弹窗
```

远程桌面只是其中一层。更实用的组合通常是：

```text
SSH:
适合看日志、跑命令、使用 tmux / zellij。

远程桌面:
适合处理 GUI、浏览器预览、权限弹窗。

Web 面板:
适合把开发机上的 dashboard 暴露给私域网。
```

## 手机远程桌面的定位

手机可以作为远程控制台，但不适合作为完整开发主力。

更合理的分工是：

```text
家庭小主机:
跑真正的任务、开发环境、自动化、同步和备份。

手机:
查看状态、处理简单命令、远程桌面点几下、临时救火。
```

远程桌面 App 可以理解成 Viewer：

```text
接收远端电脑画面
显示在手机屏幕上
把触摸、键盘、鼠标输入发回远端电脑
```

如果已经有 WireGuard 私域网，远程桌面 App 不需要承担公网中继角色。它只要连接私域 IP 即可：

```text
手机远程桌面 App
-> WireGuard 私域网
-> 10.8.0.2:5900
-> 远端电脑屏幕共享服务
```

## 同步和备份

私域网常和家庭小主机一起使用。家庭小主机可以作为常驻同步中心：

```text
代码仓库:
Git / 私有 bare repo / 托管平台

文档和知识库:
Syncthing / rsync / Git

照片、素材、项目文件:
按目录做增量同步

整机灾备:
Time Machine / 快照 / 外接盘 / NAS
```

不要把“远程访问”误解成“必须完整复制整台电脑”。更稳的目标是：

```text
关键文件可同步
关键任务可监控
关键数据可恢复
常驻节点长期在线
```

## 安全基线

私域网络不是把内网服务裸露到公网。正确做法是：公网只暴露组网入口，内部服务只在私域网里访问。

安全基线：

```text
1. 每台设备独立密钥。
2. 丢设备后立刻删除对应 peer。
3. VPS 防火墙只放行必要端口。
4. SSH 优先使用密钥登录。
5. 远程桌面、SMB、开发服务器不要裸露公网。
6. 私域服务也要设置应用层密码。
7. 定期更新 VPS、组网软件和操作系统。
8. 不把 WireGuard 默认配置成全局出口，除非明确需要。
```

要区分加密和伪装：

```text
WireGuard:
强加密、性能好、适合私域互联，但它不是伪装协议。

Reality / TLS 伪装类代理:
更适合公网代理入口的流量形态隐藏。
```

所以更清楚的分工是：

```text
WireGuard = 私人设备内网
Xray Reality = 对外代理入口
```

## 最小心智模型

```text
VPS:
云上的固定门牌号。

家庭小主机:
长期在线的内网锚点。

笔记本电脑:
在线时加入私域网的工作设备。

手机:
移动控制台。

WireGuard / Tailscale:
把自己的设备连成加密内网。

Xray / Clash / Shadowrocket:
处理代理上网、出口和分流。
```

代理系统让外网看到合适的出口；私域网络让自己的设备安全互访。两者边界清楚，系统就会更稳定、更好排错。
